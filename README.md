# RenderSaaS

پروژه‌ی نمونه‌ی یک SaaS رندرگیری معماری، با لاگین، داشبورد، تاریخچه، حساب کاربری و
جای آماده برای اتصال به API رندرگیری خودت و Stripe.

## پیش‌نیاز

- Node.js (نسخه ۱۸ یا بالاتر) — از قبل روی سیستمت نصب است.

## راه‌اندازی (فقط یک بار)

این دستورها را **داخل پوشه‌ی پروژه** و در **PowerShell** یا **ترمینال VS Code** اجرا کن:

### ۱. نصب پکیج‌ها

```
npm install
```

چند دقیقه طول می‌کشد. صبر کن تا تمام شود.

### ۲. ساخت فایل تنظیمات

فایل `.env.example` را کپی کن و اسمش را بگذار `.env`:

```
copy .env.example .env
```

(نیازی به تغییر مقادیر داخلش نیست، برای تست روی سیستم خودت همینطوری کار می‌کند.)

### ۳. ساخت دیتابیس

```
npx prisma generate
npx prisma db push
```

این دو دستور فایل دیتابیس SQLite (`dev.db`) را می‌سازند.

## اجرای پروژه (هر بار که می‌خواهی پیش‌نمایش بگیری)

```
npm run dev
```

بعد مرورگر را باز کن و برو به:

```
http://localhost:3000
```

برای متوقف کردن سرور: `Ctrl + C` در همان پنجره.

## ساختار پروژه

```
src/
  app/                → صفحات سایت (هر پوشه = یک آدرس/route)
    page.tsx          → صفحه‌ی اصلی (لندینگ)
    login/            → صفحه‌ی ورود
    signup/           → صفحه‌ی ثبت‌نام
    pricing/          → صفحه‌ی قیمت‌گذاری
    dashboard/        → داشبورد (نیاز به لاگین دارد)
    history/          → تاریخچه‌ی رندرها (نیاز به لاگین دارد)
    account/          → حساب کاربری (نیاز به لاگین دارد)
    api/              → API های سرور (پشت صحنه)
  components/         → قطعه‌های قابل استفاده‌ی مجدد (دکمه، کارت و...)
  lib/                → توابع کمکی
    render.ts         → مهم‌ترین فایل برای تو — اینجا باید API رندرگیری واقعی‌ات را وصل کنی
prisma/
  schema.prisma       → ساختار جدول‌های دیتابیس
```

## وصل کردن API رندرگیری واقعی خودت

فایل `src/lib/render.ts` هم‌اکنون به سرویس [AvalAI](https://avalai.ir) (مدل تصویری
`gemini-3.1-flash-lite-image`) وصل است. برای اجرا باید مقدار `AVALAI_API_KEY`
را در فایل `.env` خودت قرار دهی (نمونه در `.env.example`). اگر خواستی به یک
API دیگر وصل شوی، کافیست بدنه‌ی تابع `renderImage` در همین فایل را عوض کنی —
بقیه‌ی پروژه (دیتابیس، دشبورد، تاریخچه، محدودیت پلن) بدون تغییر کار خواهد کرد.

## دیدن محتوای دیتابیس (اختیاری)

```
npx prisma studio
```

این یک صفحه‌ی مرورگری باز می‌کند که می‌توانی جدول کاربران و رندرها را ببینی.

## چیزهایی که هنوز placeholder هستند

- **Stripe (پرداخت)**: دکمه‌های Subscribe فعلاً فقط یک پیام نشان می‌دهند.
  وقتی حساب Stripe و کلیدهایش را داشتی، بگو تا آن قسمت را هم کامل کنیم.
- **پروفایل عمومی کاربر** (`/u/[username]`): فیلدهای `isPublic`/`bio`/`website`
  در دیتابیس آماده‌اند ولی صفحه‌ی نمایشش هنوز ساخته نشده.

## محدودیت‌های شناخته‌شده و نکات امنیتی (قبل از production بخوان)

این پروژه یک MVP/دمو است و قبل از استفاده‌ی واقعی (production) این موارد را
در نظر بگیر:

- **Rate limiting درون‌حافظه‌ای است**: پیاده‌سازی فعلی (`src/lib/rate-limit.ts`)
  فقط برای یک instance واحد از سرور درست کار می‌کند. اگر روی چند سرور/container
  دیپلوی کنی، باید با یک storage مشترک مثل Redis (مثلاً `@upstash/ratelimit`)
  جایگزین شود.
- **بدون تایید ایمیل یا CAPTCHA**: ثبت‌نام فقط با rate limit روی IP محافظت
  می‌شود؛ برای production بهتر است تایید ایمیل یا CAPTCHA هم اضافه شود.
- **SQLite فقط برای توسعه مناسب است**: برای production به Postgres یا MySQL
  مهاجرت کن (کافیست `provider` و `DATABASE_URL` در `prisma/schema.prisma`
  عوض شود، بقیه‌ی کوئری‌های Prisma بدون تغییر کار می‌کنند).
- **عکس‌ها به‌صورت base64 در دیتابیس ذخیره می‌شوند**: برای MVP کافی است، ولی
  در مقیاس بزرگ‌تر بهتر است به یک object storage مثل S3/Cloudflare R2 منتقل شود
  و فقط URL آن در دیتابیس ذخیره شود.
- **بدون فرآیند فراموشی/بازنشانی رمز عبور**: در حال حاضر کاربر راهی برای
  بازیابی رمز فراموش‌شده ندارد.
- **`NEXTAUTH_SECRET` باید حتماً در production عوض شود** به یک رشته‌ی رندوم
  و طولانی (دستور ساختش در `.env.example` هست) و هرگز نباید commit شود.
